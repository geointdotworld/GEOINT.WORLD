GETTING STARTED

GEOINT.WORLD operates as a unified 3D geospatial intelligence terminal. This guide outlines standard viewport controls and data layer management protocols.

// CONTROL INTERFACE

  • [ L-CLICK + DRAG ]
    Rotate the globe view.
  • [ R-CLICK + DRAG ]
    Adjust camera pitch (tilt) and bearing. Essential for 3D visualizations of satellite orbits.
  • [ SCROLL ]
    Dynamic zoom level. Transitions seamlessly from orbital view down to street-level mapping.
  • [ L-CLICK (TARGET) ]
    Select any asset (plane, ship, satellite) to open its telemetry card. Displays velocity, altitude, origin, and identification data.
  • [ ONE FINGER DRAG ]
    Rotate the globe view.
  • [ TWO FINGER DRAG ]
    Adjust camera pitch (tilt) and bearing. Essential for 3D visualizations of satellite orbits.
  • [ PINCH TO ZOOM ]
    Dynamic zoom level. Transitions seamlessly from orbital view down to street-level mapping.
  • [ TAP (TARGET) ]
    Select any asset (plane, ship, satellite) to open its telemetry card. Displays velocity, altitude, origin, and identification data.

// SEARCH SYSTEM

The platform features a global, high-performance search engine capable of indexing live telemetry and static infrastructure simultaneously.

  • AUTOCOMPLETE PROTOCOL
    Search results refresh dynamically after entering at least 2 characters. The dropdown categorizes hits by type (e.g., SKY, SEA, SPACE, CABLE).
  • SEARCHABLE PARAMETERS
    You can search by ICAO/Callsign (Flights), SSVID/IMO (Ships), Node Names/IDs (LoRa), or simply enter a City or Country to geocode and fly to any location.
  • PRECISION NAVIGATION
    Selecting a search result will automatically fly the camera to the target coordinates and trigger its detailed telemetry popup.
If a red location pin remains after searching, simply click the "✕" in the search bar to clear the target marker.

// DATA LAYERS

The sidebar on the left controls the visibility of all intelligence layers. Data is grouped by domain:

HIERARCHY

  • SEA AND SKY: Moving assets (Aircraft, Vessels, Satellites).
  • COMMUNICATIONS: Static infrastructure (Radio towers, Cables) and dynamic signals (LoRa).
  • NEWS: Geopolitics and event monitoring.
  • NATURAL: Environmental hazards.
Use the ⚙ icon next to layers to access advanced filters (e.g., filtering flights by altitude or satellites by country of origin).

// LIVE FEED

The red live feed button below the toggle box shows inscriptions made from users, stored on the solana blockchain.

MECHANICS

The inscriptions seen on the live feed data are not stored on any one server. Instead, they are fetched directly from the Solana blockchain.

  • LISTENING: The client constantly scans the blockchain for inscriptions interacting with the GEOINT Memo Program.
  • DECODING: When a transaction is found, the client decodes the hex-encoded string in the SPL Memo instruction.
  • VERIFICATION: The signature is validated against the chain state to ensure the data is immutable and has been properly burned for.
  • VISUALIZATION: Validated messages appear in the feed. Clicking a feed item parses the coordinates and immediately moves the camera to the event location.

// INSCRIPTIONS

Users can permanently archive any visible telemetry data by burning $3 USD worth of $GEOINT tokens to the blockchain.

PROCESS

When you click the "Send to Blockchain" button on any asset popup, the following occurs:

  • INITIATION: A transaction is created that swaps $3 USD of SOL to $GEOINT.
  • BURN: The $GEOINT tokens are immediately sent to a burn address, reducing total supply.
  • ENCODING: The asset's telemetry (coordinates, timestamp, callsign, velocity) is encoded into the transaction memo.
  • PERMANENCE: The inscription is now immutable and permanently retrievable from any Solana node.
